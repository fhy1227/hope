## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **角色为中心** | 一角色一档；装备、背包、金币、法典绑定角色，不跨角色共享 |
| **拾取即拥有** | 进入背包的 `ItemInstance` 即计入存档；地面未拾取物不保存 |
| **显式写盘** | 关键节点调用 `PersistenceMgr.Save()` / `FlushSave()`；变更可 `MarkDirty()` 延迟合并 |
| **配置与实例分离** | 存档只存 `ConfigId` + 实例字段，不存 `ItemConfig` 全文 |
| **版本可迁移** | 存档带 `schema_version`，表结构变更时可写迁移器 |
| **参与者可扩展** | 运行时 Manager 用 `[PersistedData]` 注册，无需改 `PersistenceMgr` 硬编码 |
| **Godot 友好** | JSON（`user://`）；`ItemInstance` 经 `ItemSaveData` 互转，不走 `ResourceSaver` |

## 2. 存盘位置与文件结构

### 2.1 路径

常量见 `scripts/persistence/SaveSchema.cs`。

```
user://saves/
├── meta.json                 # 全局元数据（最后游玩栏位）
├── slot_0/
│   └── character.json        # 角色 0 完整存档
├── slot_1/
│   └── character.json
└── slot_2/
    └── character.json
```

单机默认 **3 个角色栏位**（`SaveSchema.MaxProfileSlots`）。

写盘使用临时文件 `character.json.tmp`，成功后 rename，避免半截 JSON。

### 2.2 meta.json 示例

```json
{
  "schema_version": 1,
  "last_slot_index": 0
}
```

> `settings` 音量等规划字段尚未写入 meta；当前仅持久化 `last_slot_index`。

## 3. 角色存档数据模型

### 3.1 CharacterSaveData 顶层结构

实现：`scripts/persistence/CharacterSaveData.cs`（`Hope.Persistence`）。

```csharp
/// <summary>角色局外存档根对象，对应 user://saves/slot_N/character.json。</summary>
public class CharacterSaveData
{
    public int SchemaVersion { get; set; } = SaveSchema.CurrentVersion;
    public string CharacterName { get; set; } = "";
    public long LastSavedUnixMs { get; set; }

    // ── 进度 ──
    public int Level { get; set; } = 1;
    public long Experience { get; set; }
    public int Gold { get; set; }                    // 字段已定义；参与者未接入（见 §10）
    public int DifficultyTier { get; set; }
    public string LastHubScene { get; set; } = "";
    public string LastAreaId { get; set; } = "";

    // ── 装备与经济 ──
    public List<ItemSaveData> Inventory { get; set; } = [];
    public Dictionary<int, List<ItemSaveData>> Equipped { get; set; } = [];
    public Dictionary<int, int> Materials { get; set; } = [];

    // ── 威能法典（规划）──
    public Dictionary<string, int> AspectCodex { get; set; } = [];

    // ── 任务 / Boss（规划）──
    public List<string> DefeatedBosses { get; set; } = [];
}
```

### 3.2 持久化 vs 非持久化

| 数据 | 存档 | 参与者 / 说明 |
|------|:----:|------|
| 背包 `ItemInstance` | ✓ | `InventoryManager` |
| 装备栏 | ✓ | `EquipManager` |
| 金币（角色级） | △ | `CharacterSaveData.Gold` 有字段；战斗中金币在 `RunStats.Gold`，**尚未写盘** |
| 材料、威能法典 | △ | DTO 字段已有，Manager 未接入 |
| 角色等级、难度 | △ | DTO 字段已有，未单独接入 |
| 地面 `Pickup` | ✗ | 离关清除 |
| 敌人、弹道、特效 | ✗ | 场景卸载销毁 |
| 战斗 Buff、冷却 | ✗ | 进关重新计算 |
| Demo 波次、商店临时升级 | ✗ | `RunManager` / `RunStats` 局内 |

## 4. 物品序列化

### 4.1 ItemSaveData

实现：`scripts/persistence/ItemSaveData.cs`。与运行时 `ItemInstance`（`Hope.Core`）一一对应。

```json
{
  "uid": "a1b2c3d4-...",
  "config_id": 1030,
  "count": 1,
  "item_level": 42,
  "rolled_rarity": 3,
  "affixes": [
    {
      "affix_id": "affix.might",
      "numeric_type": 2,
      "modifier_type": 1,
      "value": 0.84
    }
  ],
  "tempered_affixes": [],
  "temper_durability": 5,
  "masterwork_level": 0,
  "masterwork_bonus_indices": [],
  "socketed_gem_ids": [],
  "max_sockets": 0,
  "aspect_id": "",
  "enchant_count": 0
}
```

字段随 [装备系统-强化](装备系统-强化.md) 扩展；读档时忽略未知 JSON 键。

### 4.2 转换职责

| 方向 | 方法 | 位置 |
|------|------|------|
| 运行时 → 存档 | `ItemSaveData.FromInstance(ItemInstance)` | `scripts/persistence/ItemSaveData.cs` |
| 存档 → 运行时 | `itemSaveData.ToInstance()` | 同上 |

`ItemInstance` 保持运行时模型；**存档走 JSON**，保证可读与版本迁移。

### 4.3 装备栏与背包的引用策略

**当前：分存副本（简单可靠）**

- 装备栏、背包各存完整 `ItemSaveData` 列表。
- `Equip()` 从背包移除并写入装备栏；`Unequip()` 反向。
- 存档时各参与者 `CollectSaveData` 写回 `CharacterSaveData`，无需跨列表 Uid 引用。

### 4.4 配置热更新兼容

读档时若 `config_id` 在 `item.json` 中不存在：

- 装备类：标记为「腐化」物品，仅允许分解或丢弃，不参与穿戴。
- 材料类：按 `config_id` 保留数量，待配置恢复。

## 5. PersistenceMgr 与参与者机制

### 5.1 模块划分

```
scripts/persistence/
├── SaveSchema.cs              # 路径与 schema_version
├── CharacterSaveData.cs       # 存档根 DTO
├── ItemSaveData.cs            # 物品 / 词条 DTO
├── PersistedDataAttribute.cs  # [PersistedData] 标记参与者类
├── IPersistedDataParticipant.cs
├── PersistedParticipantRegistry.cs  # 反射收集参与者
├── PersistenceSerializer.cs   # JSON 序列化 / 反序列化
└── PersistenceMgr.cs          # Autoload 入口
```

### 5.2 调度流程

**读档**

```
PersistenceMgr.Load(slot)
  → 读 character.json，PersistenceSerializer.DeserializeCharacter
  → ActiveCharacter 载入内存
  → ApplyToParticipants(ActiveCharacter)
       → 扫描所有 [PersistedData] + IPersistedDataParticipant
       → 通过各类 static Instance 调用 ApplySaveData(data)
```

**存档**

```
PersistenceMgr.Save()
  → CollectFromParticipants(ActiveCharacter)
  → 写 schema_version、LastSavedUnixMs
  → PersistenceSerializer.SerializeCharacter → character.json.tmp → rename
```

### 5.3 PersistenceMgr API

Autoload：`scripts/persistence/PersistenceMgr.cs`。

```csharp
public partial class PersistenceMgr : Node
{
    public static PersistenceMgr? Instance { get; private set; }

    [Export] public bool DevSkipPersistence { get; set; }

    public int ActiveSlotIndex { get; private set; } = -1;
    public CharacterSaveData? ActiveCharacter { get; private set; }

    public bool CreateProfile(int slotIndex, string characterName);
    public bool Load(int slotIndex);
    public bool Save();
    public void MarkDirty();
    public void FlushSave();
    public bool DeleteProfile(int slotIndex);
    public bool HasProfile(int slotIndex);
    public int GetLastPlayedSlotIndex();

    public void ApplyToParticipants(CharacterSaveData data);
    public void CollectFromParticipants(CharacterSaveData data);
}
```

### 5.4 参与者契约

```csharp
[AttributeUsage(AttributeTargets.Class)]
public sealed class PersistedDataAttribute : Attribute { }

public interface IPersistedDataParticipant
{
    void ApplySaveData(CharacterSaveData data);   // 读档：DTO → 运行时
    void CollectSaveData(CharacterSaveData data); // 存档：运行时 → DTO
    void ClearPersistedState();                   // 删档 / 切换角色
}
```

**接入条件**

1. Autoload Manager，提供 `public static X Instance`
2. 类标注 `[PersistedData]`，实现 `IPersistedDataParticipant`
3. `project.godot` 中排在 `PersistenceMgr` **之前**
4. 变更持久化数据时调用 `PersistenceMgr.Instance?.MarkDirty()`

**已接入参与者**

| Manager | DTO 字段 | 公开方法（可选） |
|---------|----------|------------------|
| `InventoryManager` | `Inventory` | `LoadFromSave` / `ExportToSave` |
| `EquipManager` | `Equipped` | `LoadFromSave` / `ExportToSave` |

`Clear()` 仅用于 **删档 / 新角色**（经 `ClearPersistedState`）；**禁止**进战斗时调用。

### 5.5 新增参与者（速查）

完整步骤见 `.cursor/skills/persist-runtime-data/SKILL.md`。

1. `CharacterSaveData` 增加字段（或新建 `*SaveData`）
2. Manager 实现 `IPersistedDataParticipant`
3. `[PersistedData]` + Autoload 注册
4. `dotnet build` + 读档 / 存档回归测试

## 6. 存档时机

| 事件 | 动作 |
|------|------|
| 离关 / 传送回城 | `CollectFromParticipants` → `Save()` |
| 打开主菜单 / 退出游戏 | `FlushSave()` |
| 暂停菜单 → 主菜单 | `FlushSave()`（`PauseMenu`） |
| 离开 `Combat` 场景 | `FlushSave()`（`Combat._ExitTree`） |
| 装备 / 背包变更 | `MarkDirty()`；退关时 `FlushSave` 合并写盘 |
| 应用退出 | `PersistenceMgr._Notification(WMCloseRequest)` → `FlushSave()` |
| 战斗中 | 默认不每帧写盘；可定时 `Save()`（未实现） |

### 6.1 Dirty 合并

```csharp
PersistenceMgr.Instance?.MarkDirty();
// FlushSave / 离关时：if (有活动栏位) Save();
```

避免每次拾取都写盘；退关时强制 flush。

## 7. 加载流程

### 7.1 继续游戏（当前 Demo：直进战斗）

```
MainMenu「继续」
  → PersistenceMgr.Load(lastSlot)
  → ApplyToParticipants()     // 注入背包、装备等
  → ChangeScene(Combat)         // Demo；正式版可进 Hub
```

### 7.2 新游戏

```
MainMenu「新游戏」
  → PersistenceMgr.CreateProfile(slot, name)
  → 默认装备写入 DTO 并 ApplyToParticipants
  → ChangeScene(Combat)
```

### 7.3 进入战斗（目标 Hub 流程）

```
Hub 选关卡
  → PersistenceMgr.Save()           // 离城前写盘
  → ChangeScene(Combat)
  → Combat.ResetCombatState()         // 仅战斗临时状态
  → 加载 Level，生成 Player
  → EquipManager 属性加成来自读档数据
```

### 7.4 战斗结束

```
关卡完成 / 回城
  → （可选）合并 RunStats.Gold → CharacterSaveData.Gold
  → PersistenceMgr.Save()
  → ChangeScene(Hub)
```

## 8. 战斗临时状态

由 `Combat.ResetCombatState()` 清理（**不包含**背包/装备）：

| 状态 | 管理者 | 重置方式 |
|------|--------|----------|
| 波次、商店阶段 | `RunManager`（Demo） | 场景重载 / 局内逻辑 |
| 本局金币 | `RunManager.Stats.Gold` | 场景卸载即丢失；未合并到角色存档 |
| 地面 Pickup | Pickups 容器 | 清空子节点 |
| 敌人、弹道 | Enemies / Projectiles | 场景卸载 |
| 玩家实体 | Entities | 下关重新 Instantiate |
| 战斗内 RunStats 临时 Buff | Player 组件 | 重新 Init |

```csharp
// Combat.cs — 当前形态
public void ResetCombatState()
{
    // 不调用 InventoryManager.Clear() / EquipManager.Clear()
}
```

删档时 `PersistenceMgr.DeleteProfile` → 各参与者 `ClearPersistedState()`。
