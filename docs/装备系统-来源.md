## 1. 掉落来源总览

### 1.1 来源一览

| 来源 | 触发时机 | 适合刷取 |
|------|----------|----------|
| **开放区域** | 野外清怪 | 通用装备、金币、材料 |
| **地下城** | 进入副本通关 | 传奇、威能、经验 |
| **噩梦地下城** | 高难副本 | 精工材料、高 ilvl 装备 |
| **区域 Boss** | 固定刷新点 / 召唤 | 独特、专属威能 |
| **世界 Boss** | 定时 / 事件 | 独特、棱镜、稀有材料 |
| **精英怪** | 地图随机刷新 | 稀有+、MF 友好 |
| **宝箱 / 可破坏物** | 探索 | 稳定装备、材料 |
| **主城赌博师** | 回城 NPC | 补槽、追威能胚子 |
| **任务 / 赛季旅程** | 一次性或阶段奖励 | 保底独特、法典威能 |

> 纯单机：无多人事件、无玩家数量稀释掉落。所有来源由关卡配置与 `drop_table` 驱动。

### 1.2 敌人掉落倍率

按敌型区分掉落上下文（`DropContext`）：

| 敌型 | MF 加成 | 掉落率倍率 | 怪物等级加成 |
|------|:-------:|:----------:|:------------:|
| normal | 0 | ×1 | 0 |
| elite | +0.25 | ×2 | +2 |
| boss | +0.50 | ×1（保底生成） | +5 |

Boss 调用强制生成（至少 1 件装备）；普通怪过掉落门禁后掷骰。

### 1.3 Smart Loot

对齐 D4「掉落玩家能用的装备」：

- 默认约 **85%** 倾向玩家当前流派可用的槽位/武器类型。
- 槽位掷骰后，底材过滤：`level_req ≤ ItemLevel` 且 `slot_type` 匹配。
- 根据**已装备空缺槽**与**角色职业**，动态提升相应槽位权重。

### 1.4 拾取与存档

```
地面 Pickup → 玩家拾取 → InventoryManager 背包
                              ↓
                    回城 / 手动存档 / 退关
                              ↓
                    SaveManager 写入局外存档
```

未拾取的地面装备在离开区域时清除；**已入背包即永久拥有**（除非出售/分解）。

## 2. Boss 掉落与定向刷取

### 2.1 设计目标

减少「纯随机刷不到」的挫败感，支撑长线刷宝：

- 每个 Boss 绑定 **1~2 个独特物品** 或 **专属威能底材**。
- 击杀 Boss **保底至少 1 件**装备。
- 重复击杀仍有价值：材料、法典升级、金币、远古装备。

### 2.2 Boss 掉落表示例

| Boss | 推荐等级 | 主要掉落 |
|------|:--------:|----------|
| 森林守卫 | 15+ | 头盔独特、防御威能 |
| 熔炉巨像 | 30+ | 胸甲独特、进攻威能 |
| 虚空行者 | 45+ | 护符独特、资源威能 |
| 终焉化身 | 60 / 苦痛 | 神话独特（低概率） |

配置：`boss_drop.xlsx` → `item_id` 或 `drop_table` 引用 + `weight`。

### 2.3 定向刷取链

Boss 召唤材料**局外囤积**，形成可持续的刷取闭环：

```
刷野外精英 / 地下城 → 掉落「召唤核心」（存档）
  → 消耗核心挑战次级 Boss → 掉落「熔火碎片」（存档）
    → 消耗碎片挑战终极 Boss → 高概率独特 / 神话
```

材料可长期积累，玩家自主决定何时发起 Boss 战；无单局时间压力。

### 2.4 独特掉落规则

| 规则 | 说明 |
|------|------|
| 首杀保底 | 每角色首次击杀指定 Boss **必掉**其绑定独特之一 |
| 重复击杀 | 按权重随机；同独特可重复获得（用于对比词缀/送人/分解） |
| 等级门槛 | Boss 区域等级决定 `ItemLevel` 上限 |
| 远古 | 苦痛难度下 Boss 掉落默认远古子品质 |

### 2.5 固定掉落与随机管线

`drop_table.xlsx` 支持两种模式：

| `item_id` | 行为 |
|:---------:|------|
| `> 0` | 掉落指定配置物品（独特底材、材料） |
| `= 0` | 走 `EquipDropGenerator` 随机生成管线 |

## 3. 赌博系统 (Gamble)

### 3.1 基本机制

主城 **赌博师（Gambler）** NPC：

```
消耗金币 → 随机生成一件装备 → 直接进入背包
```

| 赌博目标 | 相对消耗 | 说明 |
|----------|:--------:|------|
| 武器 | 高 | 仅 `slot_type = 1` |
| 护甲 | 中 | 头盔 / 胸甲 / 靴子 |
| 护符 | 低 | `slot_type = 5` |
| 随机 | 最低 | 任意槽位 |

品质范围：普通 → 独特（独特概率极低）。

### 3.2 赌博生成

复用 `EquipDropGenerator`，独立上下文：

| 参数 | 赌博值 |
|------|--------|
| `MonsterLevel` | 当前角色等级或区域等级（取较高） |
| `MagicFind` | 固定 +0.15 |
| `ForcedRarity` | 0（随机） |
| `SmartLootChance` | 1.0（100% 命中所选槽位） |

### 3.3 使用场景

- **补槽**：某部位长期不出货时，花钱赌一件。
- **追威能**：赌传奇 → 分解进法典，长线收集全部威能。
- **追强效**：满级 / 苦痛后赌博可出远古 + 强效词缀。

### 3.4 经济平衡

```
期望成本 ≈ 同等级区域清理 15~20 分钟的金币收入
```

赌博是补充手段；野外/Boss 仍是高品质装备主来源。

## 4. 掉落生成管线

### 4.1 完整流程

```
敌人死亡 / 开箱
    ↓
匹配 drop_table 规则（区域、敌型、Boss 表）
    ↓
构建 DropContext（区域等级、怪物等级、MF、掉落率）
    ↓
【L1 门禁】drop_rate × 敌型倍率 → 未通过则不掉装备
    ↓
【L2】ItemLevelCalculator.Roll → ItemLevel
    ↓
【L3】RarityRoll.Roll → 品质（加权 + MF + 5% 升档）
    ↓
【L4】BaseItemPicker → 槽位 + 底材
    ↓
【L5】AffixPool.RollAffixes → 词条列表
    ↓
组装 ItemInstance → 地面 Pickup
    ↓
拾取 → 背包 → 存档
```

### 4.2 分层掷骰原则

| 层 | 决定内容 | MF 是否参与 |
|----|----------|:-----------:|
| L1 门禁 | 是否掉落 | 否 |
| L2 品质 | Normal ~ Unique | 是 |
| L3 类型 | 槽位 + 底材 | 否（Smart Loot 除外） |
| L4 词缀 | 词条种类与数值 | 间接（受 ItemLevel 影响） |

### 4.3 升档机制

品质掷骰后，若未达上限，**5%** 概率升一档（`UpgradeChance`）。

### 4.4 MF 递减

装备词缀、消耗品、难度加成提供的 MF 套用递减，避免后期品质泛滥：

```
effectiveMF = rawMF / (1 + rawMF × 0.5)
```

系数写入 `balance.xlsx`。

### 4.5 物品实例不可变

物品在 `Generate()` 完成瞬间，全部属性确定：

- 无「未鉴定」状态。
- 拾取后属性不变；仅强化系统（附魔/淬炼/精工）可修改指定字段。

## 5. 难度与等级的影响

### 5.1 物品等级（ilvl）

```
ItemLevel ≈ 区域等级 ± 波动
MonsterLevel = 区域等级 + 敌型加成
```

区域等级由关卡配置（`level.xlsx` / 场景元数据）定义，**不依赖** Demo 波次公式。

### 5.2 品质门槛

`quality.min_item_level` 限制低等级区域不出过高品质：

| 品质 | min_item_level |
|:----:|:--------------:|
| 普通 | 1 |
| 魔法 | 5 |
| 稀有 | 10 |
| 传奇 | 15 |
| 独特 | 20 |

### 5.3 内容阶段

| 阶段 | 角色等级 / 区域 | 解锁内容 |
|------|-----------------|----------|
| 前期 | 1–15 | 白蓝装，偶尔黄装 |
| 中期 | 16–35 | 黄装为主，紫装出现 |
| 后期 | 36–55 | 紫装常见，Boss 独特 |
| 满级 | 56–60 | 传奇为主，高难度解锁 |
| 终局 | 苦痛 I+ | 远古、神话独特、满级精工材料 |

### 5.4 难度档位

| 难度 | 怪物强度 | 掉落加成 |
|:----:|:--------:|----------|
| 普通 | ×1.0 | 基准 |
| 困难 | ×1.3 | MF +10%，掉落率 +10% |
| 苦痛 I | ×1.6 | 远古解锁，MF +25% |
| 苦痛 II+ | ×2.0+ | 独特率提升，材料加倍 |

难度选择与角色绑定存档，可随时切换（可选惩罚：部分独特仅苦痛掉落）。

## 6. 配置与实现索引

### 6.1 配置表

| 表 | 用途 |
|----|------|
| `drop_table.xlsx` | 区域、敌型、掉落率、数量、强制稀有度、固定 item_id |
| `quality.xlsx` | 品质权重、颜色、ilvl 门槛 |
| `item.xlsx` | 底材、槽位、等级需求、基础 stat |
| `boss_drop.xlsx` | Boss 绑定掉落 |
| `gamble.xlsx` | 赌博价格、槽位权重 |
| `area.xlsx`（规划） | 区域等级、解锁难度、连接 Boss |

### 6.2 模块职责

| 模块 | 职责 |
|------|------|
| `EquipDropGenerator` | 随机装备生成管线 |
| `EquipDropResolver` | 按区域/敌型结算掉落 |
| `DropContext` | 等级、MF、Smart Loot 上下文 |
| `RarityRoll` / `AffixPool` / `BaseItemPicker` | 分层掷骰 |
| `InventoryManager` | 背包（局外持久化） |
| `EquipManager` | 装备栏（局外持久化） |
| `SaveManager` | 存档读写（见 [存档方案](存档方案.md)） |
| `Pickup` | 地面拾取 → 背包 |

### 6.3 调试

```csharp
// 强制生成测试装备（开发用）
var ctx = new DropContext { MonsterLevel = 50, AreaLevel = 48, TableDropRate = 1f };
var result = EquipDropGenerator.Generate(ctx);
```

## 附录：刷宝设计原则

| 原则 | 说明 |
|------|------|
| **Boss 保底** | 重要节点必有装备，避免空手 |
| **Smart Loot** | 大部分掉落为可用装备 |
| **分层掷骰** | 品质、类型、词缀独立，易于调表 |
| **定向 Boss** | 独特有明确出处，可规划 farming |
| **赌博兜底** | 花金币补槽，非唯一来源 |
| **局外积累** | 装备、材料、法典均持久化；刷宝进度跨会话保留 |
| **区域分级** | 低等级区不出终局品质，保护成长曲线 |
