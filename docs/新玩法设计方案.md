# 方案一：共鸣系统 (Rift Synergy)

**核心创意**：装备之间不只是数值叠加，特定组合会产生"共鸣效果"——改变技能行为、产生新机制，而非简单加数值。

## 为什么选这个

当前装备系统已经实现了词条、品质、威能体系，但所有的效果都是**数值堆叠**（加生命、加速度、加伤害）。共鸣系统让"配装"产生质变，而不是量变。而且当前架构天然支持——EventBus 可监听装备变化，ConfigManager 可加一张共鸣表即可。

## 共鸣类型

### 类型 A：双装备共鸣（Slot Synergy）

特定两个装备槽位的传奇装备组合触发：

| 共鸣名称 | 需要装备 | 效果 | 视觉表现 |
|----------|---------|------|---------|
| **疾风** | 武器(任意) + 靴子(任意传奇) | 翻滚后 3s 内攻速 x1.5 | 角色拖尾残影 |
| **铁壁** | 头盔 + 护甲(任意传奇) | 格挡成功时回复 15% 生命 | 绿色光环扩散 |
| **雷暴** | 双武器(均为传奇) | 每第 5 次攻击触发一次范围闪电 | 闪电链从目标弹射 |
| **暗影** | 头盔 + 靴子(任意传奇) | 翻滚穿过敌人时造成 50% 武器伤害 | 穿过时留下紫色轨迹 |

### 类型 B：三装备共鸣（Triumvirate）

| 共鸣名称 | 需求 | 效果 |
|----------|------|------|
| **战神** | 头盔 + 护甲 + 武器(任意传奇) | 击杀精英/Boss 后全属性 +20% 持续 10s |
| **幽灵** | 武器 + 靴子 + 饰品(任意传奇) | 翻滚后隐身 1.5s，隐身期间首次攻击 x3 伤害 |

### 类型 C：专武共鸣（Named Set）

特定名称的装备组合：

| 共鸣名称 | 组合 | 效果 |
|----------|------|------|
| **影舞者套装** | 影步背心(2001) + 残影长靴(2005) | 翻滚后在地上留下持续 2s 的残影，残影自动攻击附近敌人 |
| **大地之力** | 地震之印(2025) + 泰坦践踏(2026) | 震地技能变为 2 段：第一段原地震击，1s 后第二段从天而降 |

## 实现方案

### 数据层 — 新增共鸣配置表

新增 `tools/config/synergy.xlsx`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `synergy_id` | int | 主键 |
| `synergy_name` | string | 名称 |
| `desc` | string | 描述 |
| `type` | int | 1=双装备, 2=三装备, 3=套装 |
| `required_slots` | string | 装备槽要求，如 `"1,2"`（武器+靴子） |
| `required_quality` | int | 品质门槛，4=传奇 |
| `skill_triggers` | string | 触发技能，如 `"roll"` 或 `"*"` 表示任何 |
| `effect_type` | string | 效果类型：`attack_boost`/`heal_on_parry`/`lightning_chain`/`invis` |
| `effect_value` | float | 效果数值参数 |
| `duration` | float | 持续时间(秒)，0=立即 |
| `visual_effect` | string | 视觉效果资源路径 |

运行 `python tools/export_config.py` 自动生成 `assets/config/synergy.json` 和 `scripts/config/SynergyConfig.cs`。

### 逻辑层 — SynergyManager（新增 Autoload）

```
scripts/gameplay/systems/SynergyManager.cs
```

```csharp
namespace Hope.Systems;

public partial class SynergyManager : Node
{
    // 当前激活的共鸣列表
    private List<ActiveSynergy> _activeSynergies = new();

    // 装备变化时检查共鸣
    public void OnEquipmentChanged()
    {
        _activeSynergies.Clear();
        var equipped = EquipManager.Instance.GetAllEquipped();

        foreach (var synergy in SynergyConfigManager.Instance.List)
        {
            if (CheckSynergy(synergy, equipped))
            {
                _activeSynergies.Add(new ActiveSynergy(synergy));
            }
        }

        // 触发事件
        EventBus.Instance.EmitSynergiesUpdated(_activeSynergies);
    }

    // 技能触发时检查是否有对应共鸣
    public void OnSkillUsed(PlayerActionId actionId)
    {
        foreach (var active in _activeSynergies)
        {
            if (active.Config.skill_triggers == "*" ||
                active.Config.skill_triggers == actionId.ToString().ToLower())
            {
                ApplySynergyEffect(active);
            }
        }
    }
}
```

# 方案二：命运织机 (Fate Loom)

**核心创意**：每波战斗结束后，除了商店之外，玩家还可以从 3 张"命运卡牌"中选择一张。卡牌效果是本局内生效的全局加成，且卡牌之间会相互作用、产生连锁。

## 为什么选这个

当前游戏流程是"战斗 → 商店(选升级) → 战斗"的循环，选择维度有限。命运织机增加了一个**策略分支点**，让每局的 build 走向更加多样。卡牌之间有构建组合的乐趣（类似 Slay the Spire 的遗物互动）。

## 命运卡牌分类

### 基础卡（Common，权重 60）

| 卡牌 | 效果 | 备注 |
|------|------|------|
| 钢之心 | 最大生命 +20 | 简单直接 |
| 疾风之翼 | 移速 +15% | 提升操作空间 |
| 锐利之刃 | 伤害 +10% | 通用增伤 |
| 铁壁之盾 | 护甲 +15 | 防御向 |
| 贪婪之手 | 金币获取 +30% | 经济卡 |
| 经验之书 | 经验获取 +25% | 升级加速 |

### 稀有卡（Rare，权重 30）

| 卡牌 | 效果 |
|------|------|
| 嗜血 | 击杀敌人回复 3% 生命 |
| 弹射 | 攻击有 20% 概率弹射到附近另一个敌人 |
| 连击 | 连续命中同一目标，每击伤害 +5%，最多 +50%，切换目标重置 |
| 掠夺 | 击杀敌人时 15% 概率掉落金币 |
| 荆棘 | 受到伤害时反弹 30% 给攻击者 |
| 时计 | 每波开始时获得 3s 减速时间（敌人减速 50%） |

### 史诗卡（Epic，权重 10）

| 卡牌 | 效果 |
|------|------|
| 无限 | 每 30s 下一次技能无 CD |
| 风暴 | 翻滚路径上留下持续 3s 的飓风，每秒造成 20% 伤害 |
| 共鸣增幅 | 你的共鸣效果数值 x1.5（需共鸣系统已接入） |
| 幸运 | 所有随机事件的有利概率 +15% |

### 卡牌连锁（Card Chain）

三张特定卡牌同时持有会触发额外效果：

| 连锁 | 卡牌组合 | 额外效果 |
|------|---------|----------|
| **狂战士** | 嗜血 + 连击 + 荆棘 | 生命低于 30% 时全属性 +40% |
| **掠夺者** | 贪婪之手 + 掠夺 + 经验之书 | 所有获取量 +50%（金币/经验/掉落） |

# 方案三：裂隙挑战 (Rift Challenge)

**核心创意**：战斗中偶尔会出现"裂隙"，玩家可以选择进入一个微型挑战场景。成功完成获得稀有奖励，失败则付出代价。打破了纯波次的单调性。

## 为什么选这个

当前战斗的问题是**缺乏变化**——每一波都是"周围刷怪→清掉"的循环。裂隙挑战创造了**节奏变化**和**风险决策**：进不进？值不值得冒险？而且利用了现有的基础架构（场景切换、掉落系统、战斗系统），不需要大规模重构。

## 裂隙类型

### 类型 1：黄金裂隙（每 3-5 波出现）

```
外观：金色发光裂缝
进入条件：走进裂隙按 E 交互
场景：封闭小竞技场
挑战：30s 内击杀 20 个敌人
成功奖励：大量金币 + 1 件稀有品质装备
失败惩罚：无（可安全退出）
稀有度：普通
```

### 类型 2：试炼裂隙（每 5-8 波出现）

```
外观：紫色旋转裂隙
进入条件：走进裂隙按 E 交互
场景：Boss 竞技场
挑战：击败一个精英 Boss（比普通精英更强）
成功奖励：1 件传奇品质装备 + 1 张命运卡牌
失败惩罚：退出时失去 30% 当前生命
稀有度：稀有
```

### 类型 3：混沌裂隙（每 8-12 波出现）

```
外观：红黑脉动裂隙
进入条件：走进裂隙按 E 交互（需要至少 50% 生命）
场景：混沌空间
挑战：存活 20s，持续生成强力敌人
成功奖励：1 件暗金品质装备 或 专属威能
失败惩罚：角色死亡（游戏结束）
稀有度：史诗
```

### 类型 4：宝藏裂隙（Bouns）

```
触发条件：击杀敌人时 1% 概率掉落裂隙碎片，收集 5 个碎片后激活
外观：金色宝箱旋转发光
内容：连续打开 3 个宝箱，每个宝箱三选一
奖励池：装备/金币/卡牌/消耗品
```

# 方案四：元素反应 (Elemental Resonance)

**核心创意**：为武器和技能增加元素属性（火、冰、雷、毒），不同元素之间的组合会产生独特的反应效果。不再是单纯堆伤害，而是要考虑元素搭配策略。

## 为什么选这个

这是目前游戏**最缺的一个维度**——所有伤害都是"物理伤害"，没有任何属性区分。元素系统让 Build 构筑多了一个决策层：选什么元素？怎么搭配技能和装备的元素？而且元素反应的"化学反应"感很强，视觉反馈丰富。

## 元素体系

### 4 种基础元素

| 元素 | 颜色 | 基础效果 | 获得方式 |
|------|------|----------|----------|
| **火** | 🔴 红 | 命中附加燃烧(DOT 3s) | 武器元素词条、传奇威能 |
| **冰** | 🔵 蓝 | 命中附加减速 20%，持续 2s | 同上 |
| **雷** | 🟡 黄 | 命中 15% 概率麻痹 0.5s | 同上 |
| **毒** | 🟢 绿 | 命中附加中毒(DOT 5s，可叠加 3 层) | 同上 |

### 元素反应表

| 组合 | 反应名称 | 效果 | 视觉 |
|------|---------|------|------|
| **火 + 火** | 烈焰 | 燃烧伤害 x1.5，扩散到附近 1 个敌人 | 火焰溅射 |
| **冰 + 冰** | 冻结 | 减速叠加到 60% 时长 1s | 冰冻碎裂 |
| **雷 + 雷** | 过载 | 麻痹概率提升到 40% | 电流增强 |
| **毒 + 毒** | 剧毒 | 中毒层数上限提升到 5 层 | 毒雾加深 |
| **火 + 冰** | 蒸发 | 移除燃烧和减速，造成 2 倍瞬时伤害 | 白色蒸汽爆炸 |
| **火 + 雷** | 爆炸 | 范围爆炸，半径 60px，造成 1.5 倍伤害 | 橙红色爆炸圈 |
| **火 + 毒** | 焚毒 | 燃烧和中毒合并为"焚毒"DOT，每秒伤害 x2 | 紫红色火焰 |
| **冰 + 雷** | 超导 | 范围减速 50% + 麻痹 1s | 蓝白色电网 |
| **冰 + 毒** | 凝滞 | 冻结 DOT 计时，冻结结束后造成累积伤害 | 绿色冰晶 |
| **雷 + 毒** | 腐蚀 | 降低目标护甲 20%，持续 3s | 黄绿色侵蚀 |
| **火+冰+雷** | 毁灭 | 大范围爆炸，清除全场普通敌人 | 全屏闪光 |

// ElementalDamage — 每个伤害实例携带的元素信息
public struct ElementalDamage
{
    public float BaseDamage;
    public List<ElementType> Elements;  // 可同时携带多种元素
    public float ProcChance;           // 触发概率
}

// ElementManager
public partial class ElementManager : Node
{
    // 查询两种元素组合的反应
    public ElementReaction? GetReaction(ElementType a, ElementType b)
    {
        // 查配置表
    }

    // 处理攻击命中时的元素效果
    public void ProcessElementalHit(Enemy enemy, ElementalDamage damage)
    {
        var activeElements = new List<ElementType>();

        // 1. 应用元素状态效果
        foreach (var element in damage.Elements)
        {
            ApplyStatusEffect(enemy, element, damage.ProcChance);
            activeElements.Add(element);
        }

        // 2. 检查敌人身上已有的元素状态
        var existingElements = GetStatusComponent(enemy).ActiveElements;

        // 3. 检测元素反应
        foreach (var existing in existingElements)
        {
            foreach (var newElem in activeElements)
            {
                var reaction = GetReaction(existing, newElem);
                if (reaction != null)
                {
                    TriggerReaction(enemy, reaction.Value);
                    // 消耗已有元素状态
                    GetStatusComponent(enemy).RemoveElement(existing);
                    return; // 每帧只触发一次反应
                }
            }
        }

        // 4. 记录新元素到敌人状态
        foreach (var element in activeElements)
            GetStatusComponent(enemy).AddElement(element);
    }
}
```

# 方案五：回响系统 (Echo System)

**核心创意**：每次死亡/通关后，会生成一个"回响印记"——记录你本局的关键数据（使用的技能、装备、击杀数、死亡波次等）。在后续的游戏中，可以在特定地点激活回响，获得基于之前强力 Build 的加成或挑战。

## 为什么选这个

这是一个**跨局成长**系统，但和传统的"升级天赋树"不同——它让玩家自己的历史记录成为游戏资源。"上局我用双火刀通关了，这局开局可以激活火刀回响获得加成"。这创造了一种独特的叙事感和养成沉浸感，且技术实现完全基于已有的存档系统。

## 核心机制

### 回响印记（Echo Mark）

每局游戏结束后（无论死亡还是通关），系统根据表现生成一个"回响印记"：

```
回响印记包含：
├─ 基础信息：角色等级、波次、存活时间
├─ 核心 build：使用的武器/装备/技能组合
├─ 战绩：击杀数/精英击杀/Boss击杀
├─ 评分：S/A/B/C 四级
└─ 标记类型：基于 Build 关键词
```

评分标准：

| 评分 | 条件 | 生成概率 |
|:----:|------|:--------:|
| S | 击杀 Boss 或波次 ≥ 20 | 10% |
| A | 击杀精英或波次 ≥ 12 | 25% |
| B | 波次 ≥ 6 | 40% |
| C | 低于 6 波 | 100% |

### 回响激活

在每局游戏的**第 3 波和第 8 波**，会在地图上出现"回响祭坛"，可以激活最多 2 个回响印记：

| 印记评分 | 消耗 | 效果 |
|:--------:|------|------|
| S | 无消耗 | 获得该回响的完整 Build 加成（数值 60%）+ 特殊外观 |
| A | 50 金币 | 获得该回响的核心武器（降 10 级） |
| B | 100 金币 | 获得随机一件该回响曾持有的传奇装备 |
| C | 150 金币 | 获得少量属性加成 |

### Build 关键词系统

```
回响印记自动标注关键词，匹配当前 build 时可激活特殊加成：
├─ "火焰" — 配装含火元素装备
├─ "冰霜" — 配装含冰元素装备
├─ "防御" — 生存向装备偏多
├─ "攻速" — 攻击速度类装备偏多
├─ "翻滚大师" — 翻滚系列传奇 ≥ 2 件
├─ "大地之怒" — 震地系列传奇 ≥ 2 件
```

**关键词深度激活**：如果当前局的 Build 和回响印记的 Build 有相同的关键词，则回响效果 **x1.5**。

### UI 表现

```
回响印记以"记忆碎片"形式展示：
├─ 卡片风格，半透明底色
├─ 显示：评分(S/A/B/C)、角色剪影、Build 关键词标签
├─ 悬停查看详细数据（装备列表、击杀数等）
└─ 激活后卡片燃烧消失，角色获得增益光环
```

# 综合路线图

## 优先级建议

| 优先级 | 方案 | 理由 |
|:------:|------|------|
| 🥇 | **共鸣系统** | 改动最小、与现有装备系统深度结合、产出大量配装策略 |
| 🥇 | **命运织机** | 打破波次循环单调性、决策点多、与共鸣有协同 |
| 🥈 | **裂隙挑战** | 增加节奏变化，但场景开发成本略高 |
| 🥈 | **元素反应** | 好玩但改动面较广，建议在共鸣之后接入 |
| 🥉 | **回响系统** | 创意最好但偏后期系统，适合作为"1.0 版本"的特色内容 |

## 开发顺序建议

```
Phase 1 (1周)
├─ 共鸣系统 — 基础双装备共鸣
└─ 命运织机 — 基础卡牌池

Phase 2 (1周)
├─ 裂隙挑战 — 黄金裂隙 + 试炼裂隙
└─ 元素反应 — 基础 4 元素 + 基础反应

Phase 3 (1周)
├─ 共鸣系统扩展 — 三装备 + 套装共鸣
├─ 命运织机扩展 — 稀有 + 史诗卡牌 + 连锁
└─ 裂隙挑战扩展 — 混沌裂隙 + 宝藏裂隙

Phase 4 (1周)
└─ 回响系统 — 完整实现
```

```
