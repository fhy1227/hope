## 1. 装备槽位与武器类型

### 1.1 基础槽位

Hope 目标配置为 **5 类槽位、6 个装备位**（武器可双持）：

| 槽位 ID | 类别 | 数量 | 配置来源 |
|:-------:|------|:----:|----------|
| 1 | 武器 | 2 | `equip_slot.json` |
| 2 | 头盔 | 1 | |
| 3 | 胸甲 | 1 | |
| 4 | 靴子 | 1 | |
| 5 | 护符 | 1 | |

> **扩展预留**：手套、裤子、戒指等槽位可在 `equip_slot.json` 与 `item.xlsx` 中增量添加，逻辑由 `EquipManager` 统一驱动。

### 1.2 武器类型与底材

武器底材在 `item.json` 中配置，通过 `slot_type = 1` 标识。底材示例：

| 底材 | 倾向属性 | 设计定位 |
|------|----------|----------|
| 武士刀 | 伤害 + 攻速 | 均衡近战 |
| 长剑 | 高伤害 | 慢速重击 |
| 长弓 | 伤害 + 暴击 | 远程暴击流 |
| 火枪 | 高伤害、减攻速 | 高爆发远程 |

每件武器底材的 `stat_*` 字段即其**基础数值模板**；掉落时由物品强度与稀有度进一步缩放。

### 1.3 武器固有物理特性（规划）

对齐 D4 思路：不同武器子类型携带**不可附魔更改**的固有加成，数值随物品强度缩放。

| 武器子类型 | 固有加成（示例） |
|------------|------------------|
| 斩击类（刀/剑） | +X% 暴击伤害 |
| 射击类（弓/枪） | +X% 暴击率 或 +X% 对远距敌人伤害 |
| 钝器类（锤/棍，预留） | +X% 压制/震地伤害 |

防具固有见 [§4 固有属性](#4-固有属性-inherent-stats)。

### 1.4 运行时数据结构

```
ItemInstance（局外持久化，写入存档）
├── ConfigId      → item.json 底材
├── Uid           → 实例唯一 ID
├── ItemLevel     → 物品强度
├── RolledRarity  → 掷骰稀有度
├── Affixes[]     → 随机词条列表
└── （强化字段）   → 淬炼/精工/孔位等，见强化篇
```

穿戴逻辑：`EquipManager.Equip()` → `ItemInstance.ComputeStatBonus()` → 合并到角色战斗属性。

**存档边界**：`ItemInstance` 存于玩家背包与装备栏；进出关卡不销毁，仅战斗中的临时 Buff 随场景卸载清除。

## 2. 物品品质体系

### 2.1 品质等级一览

配置表：`quality.xlsx` → `quality.json`；运行时稀有度以 `QualityConfig.Id` 为准。

| ID | 品质 | 颜色 | 随机词缀数 | 威能 | 可淬炼 | 可精工 | 可附魔 |
|:--:|------|:----:|:----------:|:----:|:------:|:------:|:------:|
| 1 | 普通 (Normal) | 灰 | 0 | — | 否 | 否 | 否 |
| 2 | 魔法 (Magic) | 蓝 | 1 | — | 否 | 否 | 否 |
| 3 | 稀有 (Rare) | 黄 | 4 | — | 是 | 是 | 是 |
| 4 | 传奇 (Legendary) | 紫 | 4 | +1 | 是 | 是 | 是 |
| 5 | 独特 (Unique) | 金 | 固定 | +1 独特威能 | 否 | 是 | 否 |

### 2.2 子品质：普通 vs 远古

高难内容（如「苦痛」难度、终局区域）掉落的装备带 **远古 (Ancestral)** 子品质：

| 子品质 | 最大物品强度 | 强效词缀 | 精工上限 |
|--------|:------------:|:--------:|:--------:|
| 普通 | 750 | 无 | 8 级 |
| 远古 | 800 | 至少 1 条 | 12 级 |

远古传奇掉落时**必定**带至少 1 条强效词缀。

### 2.3 角色成长曲线

以**角色等级**与**区域等级**共同控制掉落品质倾向（对齐 D4 满级 + 难度分层）：

| 角色等级阶段 | 掉落趋势 |
|--------------|----------|
| 1–15 | 普通、魔法为主 |
| 16–30 | 稀有占比上升 |
| 31–45 | 传奇开始出现 |
| 46–60（满级） | 传奇为主；解锁高难度 |
| 苦痛难度+ | 远古子品质；独特、神话独特 |

`QualityConfig.min_item_level` 门槛与 `ItemLevel` 联动，低于门槛的品质不参与掷骰。

## 3. 物品强度 (Item Power)

### 3.1 基本概念

物品强度（代码字段 `ItemLevel`）是衡量装备基础数值的**核心一维指标**：

```
物品强度越高 → 底材 stat 缩放越大 → 词缀数值范围越大
```

生成入口：掉落管线根据怪物等级 / 区域等级掷骰 `ItemLevel`。

### 3.2 强度来源

| 因素 | 说明 |
|------|------|
| 区域等级 | 由关卡/地下城配置，决定 ilvl 基准 |
| 怪物等级 | 通常 = 区域等级；精英/Boss 有固定加成 |
| 难度档位 | 苦痛等等级提升有效 ilvl 上限 |
| 远古子品质 | 上限提升至 800 |
| 精工 | 可突破区域上限，见 [强化篇 §2](装备系统-强化.md#2-精工-masterworking) |

### 3.3 强度 vs 词缀的选择原则

- **武器**：优先物品强度——底材 `stat_damage` 是技能伤害的基数。
- **防具 / 护符**：强度达阈值后，词缀搭配比强度更关键。

## 4. 固有属性 (Inherent Stats)

固有属性来自底材配置或槽位规则，**不可通过附魔更改**，数值随物品强度缩放。

### 4.1 底材 stat

`item.json` 中 `stat_hp / stat_damage / stat_speed / stat_crit / stat_armor` 经 `ItemStatMapping.ApplyBaseStats()` 写入属性加成。传奇品质对底材属性有额外倍率（具体系数见数值配置）。

### 4.2 槽位固有（规划）

| 装备位 | 固有属性 |
|--------|----------|
| 护符 | +X 生命 或 +X% 移速（按底材） |
| 头盔 / 胸甲 | 基础护甲 |
| 靴子 | 基础护甲 + 闪避相关（对齐玩家战斗行为系统） |
| 武器 | 见 [§1.3](#13-武器固有物理特性规划) |

### 4.3 独特物品

独特底材的固有属性**固定**，与配置表一致，不受随机词缀影响。

## 5. 词缀系统

### 5.1 词缀数量按品质分布

| 品质 | 随机词缀 | 淬炼词缀 | 威能 |
|:----:|:--------:|:--------:|:----:|
| 魔法 | 1 | — | — |
| 稀有 | 4 | +2（规划） | — |
| 传奇 | 4 | +2（规划） | +1 |
| 独特 | 固定 | — | +1 独特威能 |

### 5.2 词缀池按装备位

| 装备位 | 倾向词缀 |
|--------|----------|
| 头盔 | 生命、护甲、技能等级 |
| 胸甲 | 护甲、生命 |
| 靴子 | 移速、闪避冷却、资源消耗 |
| 护符 | 全属性加成、移速 |
| 武器 | 伤害、暴击、攻速 |

实现路径：`affix.xlsx` 增加 `slot_mask` 字段，`AffixPool` 按 `slot_type` 过滤。

### 5.3 词缀数值缩放

```
词条数值 = Lerp(min_per_level, max_per_level, random) × ItemLevel
```

同一件装备**不重复**同一 `NumericType`。

### 5.4 词缀模板（示例）

| ID | 属性 | 类型 |
|----|------|------|
| affix.vitality | MaxHealth | 固定值 |
| affix.might | Damage | 百分比 |
| affix.swiftness | MoveSpeed | 百分比 |
| affix.precision | Crit | 百分比 |
| affix.fortitude | Armor | 固定值 |

## 6. 强效词缀 (Greater Affixes)

### 6.1 基本规则

- 仅出现在**远古**子品质装备上。
- 至少 1 条，最多 4 条全为强效。
- 数值 = 同词缀普通上限的 **1.5 倍**。
- UI：特殊边框色 + 掉落音效。

### 6.2 禁忌

> **禁止对强效词缀使用附魔**——重铸会将其降为普通词缀且不可恢复。

### 6.3 获取途径

| 来源 | 说明 |
|------|------|
| 苦痛难度掉落 | 远古装备必带 |
| 主城赌博 | 满级后有几率 |
| Boss 战 | 提高远古掉落率 |

## 7. 传奇威能 (Legendary Aspects)

### 7.1 基础机制

威能是 Hope 装备的**核心自定义层**。每件传奇携带一个威能效果（如「翻滚后移速提升」），可分解收录进威能法典后反复铭印。

```
掉落传奇 → 分解 → 录入法典 → 铭印到目标装备
```

### 7.2 威能分类与可铭印槽位

| 威能类型 | 可铭印位置 |
|----------|------------|
| 进攻 (Offensive) | 武器、护符 |
| 防御 (Defensive) | 头盔、胸甲 |
| 资源 (Resource) | 护符 |
| 通用 (Utility) | 靴子、护符 |
| 移动 (Mobility) | 靴子、护符 |

### 7.3 威能强度倍率

| 装备位 | 倍率 |
|--------|:----:|
| 护符 | ×1.5 |
| 双手武器（若启用） | ×2.0（仅进攻类） |
| 其他 | ×1.0 |

### 7.4 威能与底材的关系

- **底材威能**：`item.json` 中 `type = 4/5` 的预设传奇/独特，带固定效果 ID。
- **随机威能**：掉落传奇从威能池掷骰，与底材独立。

`type = 4` 条目为**固定威能底材**；随机威能池由 `aspect.xlsx` 配置。

## 8. 独特物品 (Unique)

### 8.1 设计原则

- 词缀**固定**，不可附魔、不可淬炼。
- 独特威能**专属**，不可分解录入法典。
- 可通过 **Boss 战定向刷取**（见 [来源篇 §2](装备系统-来源.md#2-boss-掉落与定向刷取)）。
- 远古独特至少 1 条强效词缀。

### 8.2 与传奇的差异

| 维度 | 传奇 | 独特 |
|------|------|------|
| 词缀 | 随机 | 配置固定 |
| 威能 | 可分解、可铭印 | 专属，不可提取 |
| 养成 | 淬炼 + 附魔 + 精工 | 仅精工 |
| 掉落 | 广泛 | Boss / 特殊事件 |

### 8.3 神话级独特（远期）

可作为终极追求：全 Boss 低概率掉落 + 特殊材料制作保底。

## 9. 威能法典 (Codex of Power)

### 9.1 法典机制

```
分解传奇 → 威能录入法典 → 消耗材料铭印到目标装备
```

| 特性 | Hope 单机设定 |
|------|---------------|
| 进度范围 | **局外存档**（与背包、装备一同持久化） |
| 自动升级 | 分解更高数值的同类威能 → 法典中该威能等级提升 |
| 使用次数 | 录入后**无限次**铭印（每次消耗金币 + 材料） |
| 生命周期 | 随角色存档永久保留；删档时一并清除 |

> 纯单机无账号共享问题；法典即玩家个人永久收藏，与装备养成同属长线进度。

### 9.2 铭印效果

| 目标 | 结果 |
|------|------|
| 稀有 → 铭印 | 升为传奇，保留原有随机词缀，获得所选威能 |
| 传奇 → 铭印 | **覆盖**原有威能 |

### 9.3 威能获取来源

| 来源 | 说明 |
|------|------|
| 关卡首通 | 固定奖励基础威能（规划） |
| 掉落分解 | 随机威能 + 升级法典等级 |
| 赌博 | 随机传奇，间接获取威能 |
| Boss | 高品质威能底材 |

### 9.4 实现规划

| 模块 | 职责 |
|------|------|
| `AspectCodexManager` (Autoload) | 法典存档、升级、查询 |
| `aspect.xlsx` | 威能 ID、类型、效果、数值曲线 |
| 主城铁匠 UI | 铭印、分解入口 |
