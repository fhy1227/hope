# 裂隙挑战 (Rift Challenge)

> 基于 Hope 项目当前架构与代码现状
> 设计原则：可行性优先，创意其次，好玩是最终目标
> 2026年7月

---

**核心创意**：战斗中偶尔会出现"裂隙"，玩家可以选择进入一个微型挑战场景。成功完成获得稀有奖励，失败则付出代价。打破了纯波次的单调性。

## 为什么选这个

当前战斗的问题是**缺乏变化**——每一波都是"周围刷怪→清掉"的循环。裂隙挑战创造了**节奏变化**和**风险决策**：进不进？值不值得冒险？而且利用了现有的基础架构（场景切换、掉落系统、战斗系统），不需要大规模重构。

## 裂隙类型

### 类型 1：黄金裂隙（每 3-5 波出现）

```
外观：金色发光裂缝
进入条件：走进裂隙按 E 交互
场景：封闭小竞技场
挑战：30s 内击杀 20 个敌人
成功奖励：大量金币 + 1 件稀有品质装备
失败惩罚：无（可安全退出）
稀有度：普通
```

### 类型 2：试炼裂隙（每 5-8 波出现）

```
外观：紫色旋转裂隙
进入条件：走进裂隙按 E 交互
场景：Boss 竞技场
挑战：击败一个精英 Boss（比普通精英更强）
成功奖励：1 件传奇品质装备 + 1 张命运卡牌
失败惩罚：退出时失去 30% 当前生命
稀有度：稀有
```

### 类型 3：混沌裂隙（每 8-12 波出现）

```
外观：红黑脉动裂隙
进入条件：走进裂隙按 E 交互（需要至少 50% 生命）
场景：混沌空间
挑战：存活 20s，持续生成强力敌人
成功奖励：1 件暗金品质装备 或 专属威能
失败惩罚：角色死亡（游戏结束）
稀有度：史诗
```

### 类型 4：宝藏裂隙（Bonus）

```
触发条件：击杀敌人时 1% 概率掉落裂隙碎片，收集 5 个碎片后激活
外观：金色宝箱旋转发光
内容：连续打开 3 个宝箱，每个宝箱三选一
奖励池：装备/金币/卡牌/消耗品
```

### 与现有系统的集成

| 系统 | 集成说明 |
|------|----------|
| WaveManager | 波次后检查裂隙生成，裂隙期间暂停波次计时 |
| EnemySpawner | 裂隙内独立刷怪逻辑 |
| EquipDropGenerator | 裂隙奖励直接调用掉落系统 |
| FateCardManager | 试炼裂隙额外奖励命运卡牌 |
| EventBus | 新增 `RiftSpawned/RiftEntered/RiftCompleted` 事件 |
| ScenePaths | 新增裂隙场景路径常量 |
| Player | 进入裂隙时保存玩家状态，返回时恢复 |

### 裂隙视觉效果

- 使用 Godot 的 `GPUParticles2D` 创建裂隙粒子效果
- 不同类型的裂隙使用不同颜色主题
- 出现在玩家附近（20-30 格距离），地图上标记位置
- 有脉动动画和音效提示
