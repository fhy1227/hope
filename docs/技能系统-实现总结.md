## 2. 架构总览

技能系统按 Hope 三层目录划分职责：

```
┌─────────────────────────────────────────────────────────────┐
│  meta 层（局外持久化）                                        │
│  SkillManager  ←→  PersistenceMgr  ←→  CharacterSaveData   │
└───────────────────────────┬─────────────────────────────────┘
                            │ SkillState / Invest / Assign
┌───────────────────────────▼─────────────────────────────────┐
│  framework 层                                                │
│  SkillDB (Autoload) · EventBus (技能信号) · SkillSaveData    │
└───────────────────────────┬─────────────────────────────────┘
                            │ GetDefinition / 事件广播
┌───────────────────────────▼─────────────────────────────────┐
│  gameplay 层（战斗对局）                                      │
│  combat.tscn                                                 │
│    ├── SkillSystems/                                         │
│    │     ├── CooldownManager      （战斗临时）                │
│    │     ├── FuryResourceSystem   （战斗临时）                │
│    │     └── SkillCastingSystem   （施放入口 + 快捷键）       │
│    ├── UI_Hud/GameHud/SkillBar                                 │
│    └── UI_Overlay/SkillTreePanel                             │
└─────────────────────────────────────────────────────────────┘
```

**访问约定**（与 `ARCHITECTURE.md` 一致）：

- 局外任意场景：`SkillManager.Instance`、`SkillDB.GetDefinition(id)`
- 战斗内施放：`SkillCastingSystem.Instance`
- 跨模块广播：`EventBus.Instance` 技能相关信号
- **禁止** UI 直接改敌人列表；伤害走 `SkillCastingSystem` → `CombatPulse` / `Enemy.TakeDamage`

## 4. 数据模型

### 4.1 技能定义（静态）

`SkillDefinition` 核心字段：

| 字段 | 含义 |
|------|------|
| `SkillId` | 唯一 ID，如 `barb_bash` |
| `Tag` | 技能树分页：`Basic` / `Core` / `Defensive` / `Brawling` / `WeaponMastery` / `Ultimate` / `KeyPassive` |
| `MaxRank` | 最大等级（主动通常 5，终极/关键被动 1） |
| `PrerequisiteIds` | 前置技能 ID |
| `PointsRequired` | 投入本技能前需要的**全局已花费点数** |
| `EffectResource` | 引用 `SkillEffectResource`（伤害、范围、冷却、怒气消耗/产生） |
| `Enhancements` | 强化分支选项（2 选 1） |
| `TreePositionX/Y` | 技能树布局坐标 |
| `IsPassive` / `IsUltimate` | 被动 / 终极标记 |

`SkillEffectResource` 常用字段：`WeaponDamagePercent`、`ResourceCost`、`Cooldown`、`Radius`、`HitShape`、`FuryGenerated`、`KnockbackForce`。

### 4.2 玩家技能状态（运行时 + 存档）

`PlayerSkillState`：

| 字段 | 含义 |
|------|------|
| `InvestedRanks` | `skillId → 已投入等级` |
| `ChosenEnhancements` | `skillId → enhancementId` |
| `ChosenKeyPassiveId` | 已选关键被动（互斥） |
| `AvailablePoints` / `TotalPointsSpent` | 可用 / 已用技能点 |
| `SkillSlots[6]` | 快捷键绑定 |

加点规则摘要：

- 每级消耗 1 点；满级后不可再加
- 前置技能至少 1 级
- 关键被动全局只能选 1 个
- 终极技能全局只能学 1 个
- 退还时检查是否有其他技能依赖该前置

### 4.3 存档 DTO

写入 `CharacterSaveData.skill_state`（`SkillSaveData`）：

```json
{
  "available_points": 3,
  "total_points_spent": 2,
  "invested_ranks": { "barb_bash": 2 },
  "chosen_enhancements": { "barb_bash": "bash_enhance_1" },
  "chosen_key_passive_id": "",
  "skill_slots": [
    { "slot": 0, "skill_id": "barb_bash" }
  ]
}
```

新角色首次读档无 `skill_state` 时，额外赠送 **5 点**技能点便于体验。

## 5. 运行时流程

### 5.1 学习技能

```
玩家打开技能树 → 点击节点
    → SkillManager.Invest(skillId)
        → PlayerSkillState.CanInvest / Invest
        → TryAutoAssignToBar（主动技能填入第一个空槽）
        → PersistenceMgr.MarkDirty
        → EventBus.SkillLearned
```

### 5.2 施放技能

```
玩家按 Z/X/C/V/1/2（需先关闭暂停型 UI）
    → SkillCastingSystem._UnhandledInput
        → TryCastFromSlot(index)
            ├─ 槽位为空 → NotLearned
            ├─ 被动技能 → NoValidTarget
            ├─ 冷却中 → OnCooldown
            ├─ 怒气不足 → InsufficientResource
            ├─ 玩家未绑定 → NoValidTarget
            └─ 成功 → 扣怒气、开冷却、ExecuteEffect
```

`ExecuteEffect` 当前逻辑：

1. `damage = effect.GetFinalDamage(rank, Player.GetActionDamage())`
2. `SkillEffectResolver` 选取敌人；无目标时圆形/扇形走 `CombatPulse.HitCount`
3. 产生怒气写入 `FuryResourceSystem`
4. 播放 `CircleFlashEffect` 范围闪光
5. 广播 `SkillCast` / `SkillHit` / `SkillEnded`

### 5.3 玩家绑定时机

`RunManager.SpawnPlayer()` 末尾调用 `SkillCastingSystem.BindPlayer(player)`，同时触发 `SkillManager.AutoFillHotbarFromLearnedSkills()`，将已学主动技能补进空快捷键槽。

### 5.4 战斗重置

`Combat.ResetCombatState()` 清理：

- `CooldownManager.ClearAll()`
- `FuryResourceSystem.Reset()`

**不清理** `SkillManager` 加点与快捷键（局外数据）。

## 6. UI 与操作

### 6.1 快捷键映射

为避免与现有战斗行为冲突，技能栏使用以下按键（**非** 设计方案中的 Q/W/E/R）：

| 槽位 | 输入 Action | 按键 | 说明 |
|:----:|-------------|:----:|------|
| 0 | `skill_slot_1` | **Z** | |
| 1 | `skill_slot_2` | **X** | |
| 2 | `skill_slot_3` | **C** | |
| 3 | `skill_slot_4` | **V** | |
| 4 | `skill_slot_5` | **1** | |
| 5 | `skill_slot_6` | **2** | |

已有占用：`Q` 震地、`R` 聚气、`Space` 翻滚、右键格挡。

### 6.2 技能树

| 操作 | 行为 |
|------|------|
| `K` 或 HUD「技能」按钮 | 打开/关闭技能树（**打开时暂停游戏**） |
| 左键节点 | 选中并显示详情 |
| 详情「学习/升级」 | 投入 1 点 |
| 详情「重置」 | 退还该技能全部点数 |
| 右键节点 | 退还 1 级 |
| 强化按钮 | 选择强化分支（存档，效果待接入） |
| 装配下拉框 | 手动绑定快捷键槽 |
| 「重置所有技能点」 | `RefundAll` |

### 6.3 技能栏

- 底部显示 6 槽图标、等级、冷却遮罩
- 怒气条显示 `FuryResourceSystem` 当前值
- 右键槽位可清空绑定

> **注意**：技能树或背包打开时游戏暂停，此时快捷键无效；需先关闭面板再施放。

## 8. 存档与重置边界

| 数据 | 作用域 | 存档 | 进战斗重置 |
|------|--------|:----:|:----------:|
| 技能加点、强化、快捷键 | 角色级（meta） | ✅ | ❌ |
| 冷却、怒气 | 战斗临时（gameplay） | ❌ | ✅ |

遵循 `ARCHITECTURE.md`：**禁止**在 `Combat.ResetCombatState()` 中调用 `SkillManager.Clear()`。删档时由 `IPersistedDataParticipant.ClearPersistedState()` 清空。

技能点总数：`SkillPointCalculator.CalculateTotalPoints(level)` = `max(0, min(level, 50) - 1)` + 声望点（声望暂未接入，默认为 0）。

## 9. 事件总线

`EventBus` 已扩展技能相关信号：

| 信号 | 触发时机 |
|------|----------|
| `SkillLearned(skillId, newRank)` | 加点成功 |
| `SkillRefunded(skillId)` | 退还 |
| `SkillEnhancementChosen(skillId, enhId)` | 选择强化 |
| `SkillTreeReset()` | 重置全部 |
| `SkillCast(skillId, rank, position)` | 施放成功 |
| `SkillHit(skillId, target, damage)` | 命中敌人 |
| `SkillEnded(skillId)` | 效果执行完毕 |
| `SkillCooldownStarted/Ended` | 冷却开始/结束 |
| `SkillResourceChanged(current, max)` | 怒气变化 |

`SkillManager.SkillStateChanged` 为 C# 信号，供 UI 刷新使用。

## 10. 与现有战斗系统的集成

```
SkillCastingSystem
    ├─ 伤害：Player.GetActionDamage() × SkillEffectResource 倍率
    ├─ 范围：SkillEffectResolver → CombatPulse.HitCount
    ├─ 单体：Enemy.TakeDamage + ApplyKnockback
    ├─ 玩家来源：RunManager.SpawnPlayer → BindPlayer
    └─ 特效：ScenePaths.CircleFlashEffect

并行系统（互不替代）：
    ├─ PlayerActionController（翻滚/聚气/震地/格挡）
    └─ PlayerWeaponController（武器自动攻击）
```

技能与 `PlayerAction` **独立调度**；未来若技能需要锁移动/无敌，应协调 `IPlayerAction` 的 `BlocksMovement` / `GrantsInvincibility`，或扩展 `SkillCastingSystem` 状态检查。

## 11. 已知限制与后续路线

### 11.1 当前限制

1. **强化分支**仅存档与 UI，不改变 `SkillEffectResource` 数值  
2. **被动 / 关键被动**可加点，无战斗内效果管线  
3. **引导技**（如旋风斩）一次施放即结算，无持续 channel  
4. **防御技**（减伤、嘲讽）无 Buff 系统支撑  
5. **伤害公式**为简化版，未接装备词缀乘区、暴击/压制/易伤  
6. **技能图标**未配置，`TextureRect` 显示为空  
7. **技能树连线**未绘制，仅节点布局  
8. 配置维护在 `SkillCatalog.cs`，未走 xlsx 工具链  

### 11.2 建议后续阶段（对齐设计文档）

| 阶段 | 内容 |
|------|------|
| 阶段二 | 完整 `DamageCalculator`、`EffectApplicator`、Buff/控制、引导技、抛射物 |
| 阶段三 | `PassiveSystem`、强化修饰符接入、职业专精 |
| 阶段四 | 技能编辑器、`.tres` 工作流、`skill.xlsx` 配置表 |

## 12. 扩展指南

### 12.1 新增技能（代码方式）

在 `SkillCatalog.cs` 中调用 `CreateSkill(...)` 并 `db.Register(...)`，或新增 `RegisterXxxSkills` 方法后在 `SkillDB._Ready` 中调用。

设置 `TreePositionX/Y` 控制技能树布局；`prerequisiteIds` / `pointsRequired` 控制解锁条件。

### 12.2 新增技能（资源方式）

1. 在编辑器创建 `SkillDefinition` Resource  
2. 保存到 `res://data/skills/<职业>/<名>.tres`  
3. `SkillDB` 启动时自动加载（无需改代码，除非新职业需单独 Catalog）

### 12.3 新增职业资源类型

参考 `FuryResourceSystem` 为法师/游侠等实现 `ResourceSystem` 子类，在 `SkillCastingSystem` 中按职业切换消耗检查。

### 12.4 接入装备 / 威能

- 局外永久加成：扩展 `DataModifierComponent` 或 `CharacterSaveData`  
- 技能行为变化：仿 `IChargeReleaseEffect` 为 `ISkillEffectModifier` 注入 `SkillCastingSystem`  
- 触发被动：订阅 `EventBus.SkillHit` / `SkillCast`

### 12.5 新增 UI 入口

战斗 UI 挂到 `combat.tscn` 既定 `CanvasLayer`（见 `UiLayers`）；局外 Hub 技能入口挂 `scenes/meta/`，通过 `SkillManager.Instance` 读写，勿直接操作战斗节点。
