# 回响系统 (Echo System)

> 基于 Hope 项目当前架构与代码现状
> 设计原则：可行性优先，创意其次，好玩是最终目标
> 2026年7月

---

**核心创意**：每次死亡/通关后，会生成一个"回响印记"——记录你本局的关键数据（使用的技能、装备、击杀数、死亡波次等）。在后续的游戏中，可以在特定地点激活回响，获得基于之前强力 Build 的加成或挑战。

## 为什么选这个

这是一个**跨局成长**系统，但和传统的"升级天赋树"不同——它让玩家自己的历史记录成为游戏资源。"上局我用双火刀通关了，这局开局可以激活火刀回响获得加成"。这创造了一种独特的叙事感和养成沉浸感，且技术实现完全基于已有的存档系统。

## 核心机制

### 回响印记（Echo Mark）

每局游戏结束后（无论死亡还是通关），系统根据表现生成一个"回响印记"：

```
回响印记包含：
├─ 基础信息：角色等级、波次、存活时间
├─ 核心 build：使用的武器/装备/技能组合
├─ 战绩：击杀数/精英击杀/Boss击杀
├─ 评分：S/A/B/C 四级
└─ 标记类型：基于 Build 关键词
```

评分标准：

| 评分 | 条件 | 生成概率 |
|:----:|------|:--------:|
| S | 击杀 Boss 或波次 ≥ 20 | 10% |
| A | 击杀精英或波次 ≥ 12 | 25% |
| B | 波次 ≥ 6 | 40% |
| C | 低于 6 波 | 100% |

### 回响激活

在每局游戏的**第 3 波和第 8 波**，会在地图上出现"回响祭坛"，可以激活最多 2 个回响印记：

| 印记评分 | 消耗 | 效果 |
|:--------:|------|------|
| S | 无消耗 | 获得该回响的完整 Build 加成（数值 60%）+ 特殊外观 |
| A | 50 金币 | 获得该回响的核心武器（降 10 级） |
| B | 100 金币 | 获得随机一件该回响曾持有的传奇装备 |
| C | 150 金币 | 获得少量属性加成 |

### Build 关键词系统

```
回响印记自动标注关键词，匹配当前 build 时可激活特殊加成：
├─ "火焰" — 配装含火元素装备
├─ "冰霜" — 配装含冰元素装备
├─ "防御" — 生存向装备偏多
├─ "攻速" — 攻击速度类装备偏多
├─ "翻滚大师" — 翻滚系列传奇 ≥ 2 件
├─ "大地之怒" — 震地系列传奇 ≥ 2 件
```

**关键词深度激活**：如果当前局的 Build 和回响印记的 Build 有相同的关键词，则回响效果 **x1.5**。

### UI 表现

```
回响印记以"记忆碎片"形式展示：
├─ 卡片风格，半透明底色
├─ 显示：评分(S/A/B/C)、角色剪影、Build 关键词标签
├─ 悬停查看详细数据（装备列表、击杀数等）
└─ 激活后卡片燃烧消失，角色获得增益光环
```

## 实现方案

### 数据层

新增存档数据结构：

```csharp
// CharacterSaveData.cs 新增
public List<EchoMarkData> EchoMarks = new();

// EchoMarkData
public struct EchoMarkData
{
    public string MarkId;            // 唯一标识
    public int Rating;               // 4=S, 3=A, 2=B, 1=C
    public int WaveReached;          // 到达波次
    public int EnemiesKilled;
    public int ElitesKilled;
    public int BossesKilled;
    public string BuildKeywords;     // 逗号分隔
    public List<int> KeyItemIds;     // 关键装备 ID 列表
    public int PlayerLevel;
    public DateTime CreatedAt;
    public bool IsActive;
}
```

最大存储 20 个印记，超出时删除最旧的 C 级印记。

    public float GetKeywordBonus(string playerKeywords, string markKeywords)
    {
        var playerTags = playerKeywords.Split(',');
        var markTags = markKeywords.Split(',');

### 与现有系统的集成

| 系统 | 集成方式 |
|------|----------|
| 存档系统 | EchoMarks 数组存入 `CharacterSaveData` |
| RunManager | 游戏结束时调用 EchoManager.GenerateEchoMark |
| RunStats | 激活回响时叠加加成到 RunStats |
| WaveManager | 第 3/8 波时触发生成祭坛 |
| EnemySpawner | 祭坛出现在地图随机位置 |
| U Hud | 波次提示时显示"回响祭坛已出现" |
| EquipManager | 激活装备类回响时直接生成对应装备 |
