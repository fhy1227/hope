## 1. 核心概念

### 1.1 术语表

| 术语 | 缩写 | 说明 |
|------|------|------|
| Item Type | — | 物品的实际类型（Rune、Jewel、Charm、Shortsword 等），不受 MF 影响 |
| Item Quality | — | 物品品质（Unique、Set、Rare、Magical、Normal），受 MF 影响 |
| Quality Level | Qlvl | 每种基础护具/武器类型的固定等级数字，同类物品 Qlvl 恒定 |
| Item Level | Ilvl | 变动的数字，取决于掉落怪物的 Mlvl 或 NPC 生成时的角色等级 |
| Monster Level | Mlvl | 怪物等级（普通 1-90，Champion/Boss 有等级奖励） |
| Affix Level | Alvl | 魔法前缀/后缀的等级，决定何种怪物能掉落该词缀 |
| Treasure Class | TC | 财宝阶层，一组相近物品的集合，用于决定物品生成 |
| Magic Find | MF | 寻宝魔法，影响物品品质（不影响类型） |

### 1.2 等级体系关系

```
Mlvl (怪物等级)
    └→ Ilvl (物品等级)  ← 通常 Ilvl = Mlvl
            └→ 结合 Qlvl (类型等级) → Alvl (词缀等级)
```

## 2. TC 系统：TreasureClass 全解析

### 2.1 文件结构

TreasureClass.txt 是掉落系统的"幕后剧本"。每只怪物被击败后，游戏引擎按照这张表查找所属"剧目"，掷骰选场景，一路递归直到把具体战利品丢到地面。

### 2.2 字段详解

| 列名 | 作用 | 要点说明 |
|------|------|----------|
| `Treasure Class` | 行标识 | 怪物、宝箱或环境效果引用的名称 |
| `Group` | 分组号 | 与怪物等级配合，自动晋级到更高 TC |
| `Level` | 宝藏类等级 | 影响自动升级上限 |
| `Picks` | 掉落次数 | **正数** = 最多随机次数；**负数** = 强制掉落固定次数 |
| `Unique/Set/Rare/Magic` | 质量权重 | 以 1024 分制，数值越大对应品质概率越高 |
| `NoDrop` | 空手权重 | 值越高空手概率越大，可被 `players X` 稀释 |
| `Item1~Item10` | 候选条目 | 可指向单个物品，也可指向另一宝藏类 |
| `Prob1~Prob10` | 权重 | 与 `NoDrop` 一起归一化为概率分布 |

### 2.3 TC 的分级结构

武器和防具各自有 **30 个 TC**，从 3-90，每隔 3 升一级：

```
TC 3  → TC 6  → TC 9  → ... → TC 87  → TC 90
(Qlvl 1-3)                         (Qlvl 87-90)
```

**Group 与 Level 的双重限制**：例如 `weap78`、`armo78` 属于高阶装备组，只有当怪物等级 ≥ 78 时才会被选中。

## 3. 物品产生流程

### 3.1 整体流程

```
怪物死亡/箱子打开
    ↓
检测是否掉落物品 (受玩家数量影响，MF 不参与)
    ↓
选定 TC 确定物品类型 (MF 不参与)
    ↓
检测 Item Quality (Unique → Set → Rare → Magical → Superior → Normal → Low)
    ↓
检测 Ethereal (5%)
    ↓
检测 Sockets (仅 Normal/Superior)
    ↓
物品生成
```

### 3.2 关键原则

> **一件物品在生成的那一瞬间，其类型和一切属性都被决定了。** 对于未鉴定物品，任何人何时鉴定都不会改变其属性。

> **所有物品状态在产生/掉落时就被决定**，杀死怪物的玩家的 Clvl 对掉落过程不产生任何影响。

## 4. 物品类型决定阶段

### 4.1 TC 递归掷骰算法

```
1. 选定顶层宝藏类（取自怪物表的 TreasureClass1~3 字段）
2. 计算总权重：NoDrop + ΣProbX
3. 掷骰：随机数落在 NoDrop 区间 → 本轮空；否则命中某条 ItemX
4. 判断是否叶节点：
   - ItemX 是具体物品 → 流程结束
   - ItemX 指向另一宝藏类 → 以它为新根重复步骤 2-4
5. 重复 Picks 次数
```

### 4.2 怪物 TC 选择概率

每一个怪物都有最高能掉落的 TC 等级限制。越高的 TC 等级被选择的几率越小：

> 例如 Hell Baal 能从 Armor TC 87 中掉落物品，但仅有 **0.0010116%** 的几率。这就是为什么你总是得到低等级物品比高等级的多得多。

当最高 TC 选择失败时，游戏随机检测下一级 TC，以此类推，直到某一 TC 成功。

### 4.3 掉落率处罚

部分物品类型有掉落率处罚：

| 物品类型 | 掉落率（相对普通物品） |
|----------|----------------------|
| 普通物品 | 1 (基准) |
| Asn Claws (刺客爪) | 2/3 |
| 角色专用物品 | 1/3 |
| Rods (Wands/Staves/Scepters) | 1/3 |

> **计算公式示例**：一个 Weapon TC 含 6 件物品（4 件普通、1 件 Claw、1 件 Amazon Spear），则 4 件普通物品各有 3/15 机会，Claw 有 2/15，Spear 有 1/15。

## 5. 物品品质决定阶段

### 5.1 品质检测顺序

```
Unique → Set → Rare → Magical → Superior → Normal → Low Quality
```

品质检测类似于 TC 检测：**最先检测发生可能性最小的品质**，以此类推。

### 5.2 品质降级规则

**情况 A**：成功选定 Unique 品质，但：
- 不存在该 Unique 物品
- 怪物的 Mlvl 不够高
- 同一件 Unique 已在当前游戏中掉落过

→ 降级为 **3 倍耐久度的 Rare** 物品

**情况 B**：成功选定 Set 品质，但不存在该 Set 物品或 Mlvl 不够

→ 降级为 **2 倍耐久度的 Magical** 物品

### 5.3 Boss 品质保底

Bosses 总是（至少）会掉落一件物品，除非该物品是 Rune 或 Gem，否则其**最低为 Magical 品质**。

### 5.4 特殊物品的品质限制

| 物品类型 | 品质限制 |
|----------|----------|
| Jewels | 仅检测 Rare / Magical（跳过 Set/Unique） |
| Charms | 永远是 Magical 品质 |
| Runes / Keys / Potions / Scrolls | 不进行品质检测 |

## 6. NoDrop 机制与多人游戏影响

### 6.1 基本规则

当杀死普通怪物时，总有很大几率选择 "No Drop"（不生成物品）。宝箱、瓮、桶等也是如此。

### 6.2 玩家数量对 NoDrop 的影响

计算公式基于"有效玩家计数"：

| 条件 | 基数加成 |
|------|----------|
| 杀死怪物的玩家 | 1 |
| 同场景结盟玩家（每人） | +1 |
| 不同场景结盟玩家（每人） | +1/2 |
| 未结盟玩家（每人，无视场景） | +1/2 |

> **`players 8` 会把 NoDrop 权重指数衰减**，对普通白怪影响最大，对 Boss 类影响较小。

## 7. MF 寻宝魔法的作用机制

### 7.1 MF 的适用范围

> **MF 只影响 Item Quality（物品品质），不影响 Item Type（物品类型）。**

### 7.2 MF 递减公式

MF 值越高，实际收益递减越明显：

| 原始 MF | 实际 Unique MF | 实际 Set MF | 实际 Rare MF |
|---------|----------------|-------------|--------------|
| 100% | 100% | 100% | 100% |
| 250% | 181% | 206% | 215% |
| 400% | 229% | 283% | 300% |
| 800% | 288% | 391% | 422% |
| 1000% | 299% | 416% | 450% |

> **关键结论**：MF 从 250% 提升到 1000%（4 倍），仅增加约 102% 的 Unique 效果。**更高的 MF 意味着更多的 Unique/Set，但性价比急剧下降。**

### 7.3 MF 的实践经验

> 在不大幅降低角色能力的前提下尽可能增加 MF 值。如果为了增加 10% Unique MF 而使效率降低一半，原本可以做 2 次以上的 MF Run，反而得不偿失。

## 8. 暗金与绿色套装掉落规则

### 8.1 掉落条件

掉落 Unique/Set 物品需要同时满足：

1. **怪物有足够高的 Mlvl** ≥ 该 Unique/Set 物品的 Qlvl
2. **通过了 TC 检测**（怪物能从对应的 TC 中掉落该基础物品类型）
3. **通过了品质检测**（Unique/Set 品质被选中）

### 8.2 Unique 每个游戏仅限一件

> 对于同一件 Unique 物品，每个游戏中仅能找到一次。角色身上正在使用的不会影响掉落。

这意味着：
- 如果你想刷 SoJ，**杀等级 39-52 之间的怪物最有效**
  - 因为它们不可能掉落 Dwarf Star 或 Raven Frost，排除竞争
  - Normal Diablo (Mlvl 40) 掉的 Unique Ring 有 **1/31** 几率是 SoJ
  - Normal Baal (Mlvl 60) 掉的 Unique Ring 仅 **1/51** 几率是 SoJ

### 8.3 Rare/Unique/Set 饰物的概率计算

```
某特定物品的掉落几率 = 该物品的 Rarity / 怪物能掉落的所有同类物品的 Total Rarity
```

## 9. 词缀选择算法

### 9.1 Alvl 计算（核心公式）

```
1. if (Ilvl > 99) then { Ilvl = 99 }
2. if (Qlvl > Ilvl) then { Ilvl = Qlvl }
3. if (magic_lvl > 0) then { Alvl = Ilvl + magic_lvl }
4. else
   {
     if (Ilvl < (99 - Qlvl / 2))
       then { Alvl = Ilvl - Qlvl / 2 }
       else { Alvl = 2 * Ilvl - 99 }
   }
5. if (Alvl > 99) then { Alvl = 99 }
```

> **关键点**：Circlets 拥有 `magic_lvl > 0`，使它们能获得 Alvl 奖励，因此 Circlets 可以出更高的词缀。

### 9.2 词缀选择的特点

词缀选择和 TC 选择最大的不同：**不存在高等级/低等级词缀的比重差别**。

> 一个 Mlvl 50 的怪物掉落等级 48 词缀和等级 14 词缀的可能性基本相同（不考虑某些词缀的权重奖励）。

## 10. 孔洞生成规则

### 10.1 天然孔洞

- 约 **1/3** 的 Normal / Superior 物品出现天然孔洞
- Shields、全部 Weapons (除投掷)、Body Armor、Helms 可以有孔
- 孔数取决于 **物品 Ilvl**（基于怪物/箱子等级）

孔数突变点：

| Ilvl 范围 | 可能孔数 |
|-----------|----------|
| < 25 | 最少 |
| 25 - 39 | 中等 |
| ≥ 40 | 最大可能孔数 (MNS) |

### 10.2 Larzuk 打孔任务

| 物品品质 | 打孔结果 |
|----------|----------|
| Normal / Superior | 打出 MNS（最大孔数） |
| Magical | 随机 1-2 孔 |
| Unique / Set / Rare / Crafted | 只能打出 1 孔 |

> Larzuk 无法在已有孔的物品上添加孔，也无法在不能有孔的物品上打孔（如投掷武器、靴子、手套等）。

## 11. Ethereal 无形属性

### 11.1 基本规则

- 全部物品约 **5%** 几率出现 Ethereal
- **永远不会出现在 Set 或 Crafted 物品上**
- 耐久度降低 45%（即只有正常的 55%）
- STR / DEX 需求各减 10 点

### 11.2 耐久度消耗率

| 防具部位 | 被击中的耐久消耗概率 |
|----------|---------------------|
| Armor | 23.1% |
| Shield | 23.1% |
| Helm | 15.4% |
| Gloves | 15.4% |
| Belt | 11.5% |
| Boots | 11.5% |

> **Ethereal 物品穿在 Pet 身上不会损失任何耐久度。**

## 12. NPC 售卖与赌博

### 12.1 NPC 售卖

NPC 售卖物品的等级公式：

```
Ilvl = Clvl + 5
```

Normal 难度各 Act 封顶：

| Act | 最大 Ilvl |
|-----|-----------|
| Act 1 | 12 |
| Act 2 | 20 |
| Act 3 | 28 |
| Act 4 | 36 |
| Act 5 | 45 |

Exceptional / Elite 升级概率：

```
Exceptional: (Ilvl - Qlvl) / 2%
Elite: (Ilvl - Qlvl) / 4%
```

99 级角色的最高几率：

| 难度 | Exceptional | Elite |
|------|-------------|-------|
| Nightmare | 10.7% | — |
| Hell | 18.3% | 2.7% |

### 12.2 赌博

赌博时的最大词缀等级：

```
最大 Alvl = Clvl - 5 到 Clvl + 4 (随机)
```

> **角色等级越高，越可能看到 Exceptional/Elite 物品。Clvl 至少需要约等于目标 Elite 物品的 Qlvl。**

## 13. 案例：女伯爵符文掉落

### 13.1 双重 TC 结构

Countess 的特殊之处在于她拥有**双重宝藏类**：

```
Countess (H)
  Picks = -3        ← 负数表示强制掉落 3 次
  ├→ Countess Rune  ← 2 次强制掉落符文
  └→ Act 1 (H) Equip A ← 1 次常规装备池
```

### 13.2 符文范围

```
Countess Rune 行:
  Item 列出 El → Ist
  结合怪物等级决定上限为 Lo (28#)
```

### 13.3 掉落示例

```
Picks 第 1 次: 命中 Countess Rune
    → 掷骰选择：Ist (1/??) 或其他低符文
Picks 第 2 次: 同上
Picks 第 3 次: 命中 Act 1 (H) Equip A
    → 掉落黄装/蓝装/灰装等
```

### 13.4 MOD 修改示例

让女伯爵额外掉落"苦难之钥"：

```csv
Treasure Class,Group,Level,Picks,Unique,Set,Rare,Magic,NoDrop,Item1,Prob1
Countess (H),4,82,-3,500,800,1024,1024,0,KeyDrop,4
KeyDrop,0,0,1,0,0,0,0,0,KeyOfDistress,1
```

## 14. 游戏设计启示

### 14.1 从 D2 掉落系统学到的设计原则

| 原则 | 说明 | 应用建议 |
|------|------|----------|
| **分层检测** | 类型 → 品质 → 词缀，逐层独立 | 掉落设计应层次分明，避免一次随机决定所有属性 |
| **概率递减** | 高价值物品概率极低，且逐级下降 | 保证玩家永远有"盼头"，但不会过度泛滥 |
| **MF 收益递减** | 避免极端堆叠单一属性 | 为属性堆叠设置合理的边际收益曲线 |
| **类型独占竞争** | 高等级物品之间互相稀释概率 | 控制池子大小，避免过多同类型物品互相稀释 |
| **保底机制** | Boss 保证至少 Magical 品质 | 为重要节点设置保底，减少空手而归的挫败感 |
| **递归 TC** | 一张表管理多层掉落逻辑 | 用树状结构管理配置，清晰且易维护 |

### 14.2 常见陷阱

| 陷阱 | 后果 | 解决方法 |
|------|------|----------|
| 权重归一化错误 | NoDrop 意外暴涨 | 新增条目后检查权重总和 |
| 负数 Picks + 空权重 | 引擎崩溃 | 务必保留一个兜底项 (如 junk) |
| 自动晋级意外 | 全图掉不该掉的物品 | 自定义 TC 不要放进自动生成序列 |
