# 2026-06-20 工作日志

## 配置表导出工具 (export_config.py)

- 创建了 `tools/export_config.py` — 基于 item.xlsx 模板格式的配置导出工具
- 功能：读取 `tools/config/` 下所有 .xlsx 文件，导出 JSON 到 `assets/config/`，生成 C# 配置类到 `scripts/config/`

### 模板格式支持
- Row 1-2: 字段名 + 类型 (int/string/float/bool)
- Row 3-4: 标签 (ignore, text, comma, table) + 模式 (!mode, client)
- Row 5: 注释
- Row 6+: 数据行

### 特性
- 自动跳过 `ignore` 列和 `~$` 临时文件
- `comma` 标签 → 导出为数组 (JSON + C# int[])
- `table` 标签 → 保留为字符串
- 跳过 `!` 开头的工作表
- 同时输出 `_dict` (key-value 查找) 和 `_list` (遍历) 格式

### C# 架构重构
- 配置类改为纯数据类 (POCO)，实现 `IConfigData` 接口
- 每个配置类含 `FromDict(GodotDictionary)` 反序列化方法
- 统一 `ConfigManager` 管理所有配置加载/缓存/查询
  - `LoadAll()` 扫描 `res://assets/config/*.json` 自动加载
  - `GetAll<T>()` 返回泛型列表
  - `Get<T>(key)` 按 ID 查询单条
  - `GetList(tableName)` / `GetDict(tableName)` 原始数据访问
- 移除旧的每表独立 Manager 模式

## @table 格式升级：string → 结构化 jsonData

- 修改 `export_config.py`:
  - `parse_value` 中 `@table` 标签改为解析 `{{k1,v1},{k2,v2}}` → `[[k1,v1],[k2,v2]]` 嵌套数组
  - 自动推断元素类型（int / string）
  - `export_csharp_config` 中 `@table` 字段 C# 类型改为 `GodotArray`
  - `FromDict` 支持兼容：新版 JSON 数组 + 旧版字符串兜底
